最近在学习java并发编程，学习java并发编程实战这本书，前几章都在介绍基础知识非常重要，但是总是感觉还没完全透彻的理解，所以先从基本应用入手，最后在对基础知识进行回顾。今天主要学习并发容器的ConcurrentMap类以及解决的问题。
在早期的jdk中，就提供了一些同步容器类，比如Hashtable,Vector，还有一个Collections.synchronizedXXX可以把非线程安全的容器转为线程安全的容器。这些同步容器的实现都是基于synchronzied来实现的，可以打开Hashtable的实现可以看到它实现Map的接口，然后看下它的put方法：

可见就是通过在方法块中使用synchronzied来实现的。也没有相关复合操作的相关接口。

基础知识

ConcurrentMap继承自Map，在Map相关接口上增加了如下接口：
V putIfAbsent(K key,V value):如果当前key已存在则返回当前key的值，否则增加到容器中。这个接口如果用Map的接口相当于：

之所以ConcurrentMap中增加这个接口是因为在多线程情况下，在业务代码中实现以上代码是有风险的，除非在这个判断写在同步块中，但是如果写在同步块中，又会影响系统吞吐量。在ConcurrentMap的实现中对锁进行了更加细粒度的控制，来减少对吞吐量的影响。在深入的原理我们后面在探讨。
boolean remove(Key key,V value):如果当前key已存在并且当前的值等于容器中得值，则删除这条值。如果使用map中得接口如下：

增加这个方法的原因和上一个方法一样。也是一个复合操作。
还有两个方法：
boolean replace(Key key,V oldValue,V newValue)：如果存在并且等于oldValue则替换；
boolean replace(Key key,V value)；:如果存在则替换。
以上相同的特点都是原子性的复合操作。
