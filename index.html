Posted on 2016-06-24 | In Archive |

为什么需要版本管理

在多模块的项目中，版本管理是一种特别重要的环节，版本管理混乱将直接导致项目的失控。
在项目发展的初期，代码是集中管理的，往往面临以下问题：

  • 业务的快速发展并且不断变化，代码也在不断的膨胀
  • 由于人力、时间等各个方面的原因，现阶段无法做到子系统的拆分
  • 团队人员在快速增加，每一次修改都可能影响到很多历史模块的

对于这样的系统，可以先做到Jar包拆分，把每个模块拆分成独立的jar包，这时jar包的管理就需要有版本的管理。

Java并发编程学习之BlockingQueue

Posted on 2015-04-13 | In java |

在并发编程中，经常会遇到如下场景，前半部分为批量操作，比如我们要到数据库中或者远程api获取一批数据，另一部分是比较耗时的计算操作。如果使用单线程的实现，就需要对批量获取的数据进行循环然后调用非常耗时的计算操作。这种实现在数据量非常大得时候可能需要数天才能完成操作。并且单线程也没用充分利用机器的CPU资源。一方面无法发挥多核CPU的优势，另一个方面在获取批量数据时线程处于阻塞状态，也浪费了线程的执行效率。以上描述的问题，其实是一个经典的生产者消费者的问题。解决以上问题，可以使用一个线程拉取数据，另一些线程做计算操作。提高CPU的利用效率，减少执行时间。这是就需要一个容器作为一个缓冲区来以此来支持这种异步操作。

Git分支管理

Posted on 2015-04-11 | In Git |

在来到新公司之后，使用Git作为源代码版本控制管理的工具，虽然一直在用git来管理自己的代码，但是还一直没有使用过分支管理。在以前的工作中，也是用过svn的分支管理，但是那种分支管理使用起来非常不方便，新建分支和切换分支都非常不方便。在svn里基本很少人愿意使用分支管理。但是Git不一样，Git可以很方便的来管理分支。所以在Git中分支管理是一个非常常用的一个功能。

为什么要使用分支管理

在一个项目开发周期，假设有一个任务，你需要两周完成，但是你在第一周的时候提交了你的50%的代码，因为代码还没写完，导致当前代码库里的代码是编译不能通过的状态，这时就影响了别人的开发。或者其他已开发好的模块希望尽快上线，这时也不能上线了。如果自己建立一个分支，自己编写的代码只提交到自己的分支，和主分支分离，这样就不会相互影响了。
